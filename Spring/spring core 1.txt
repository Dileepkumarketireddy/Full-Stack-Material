What is Framework ???
=======================

-> Framework means semi-developed software.
-> Frameworks provides some common logics which are required for application development

	Ex: Hibernate, Struts, Spring etc,..

-> If we use framework in our application development then we need to focus only on business logic
   
	(frameworks will provide common logics required for our application)


Hibernate -> 
-----------
Its a ORM frame work [Object relational mapping] used to develop ONLY persistance layer of the application.// >> DATA JPA >>

Struts -> 
---------
It is a web frame work , & used to develop only web layer of the application. but now it is outdated.


Spring frame work -> 
-----------------
-> It is an application development framework. 
-> we can develop entire end to end application by using this spring frame work.
-> means all the layers of application we can develope with this frame work. 
->Entire project can be developed by using this. [ not only web & persiatance all it can devolep ]

				
Weblayer = Controller classe [ Struts ]
Business = Service clase     [ JEE concepts ]
Persistance = DAO [DATA ACCESS OBJECT] class [CRUD operations ] logic which is responsible to communicate with the data base [ hibernate ]

===============================================
Multiple Layers in Spring frame work|
===============================================

1) Controller/web layer
2) service layer
3) Repository layer

1) Controller layer
-------------------------

-> It acts as a front controller for a java application.
-> based on the incoming request it will executes the corresponding logic present in the service layer & Returns the output to the UI.

2) Service/Business layer
------------------------
-> it contains the core business logic of each and every incoming request.

-> bases on the request it executes logic and returns the response to the contoller layer.

3) repository/Persistance layer
--------------------------------

-> it is used to establish a connection between and the java application and the data base.

-> to develop this repository or persistance layer we can use hibernate or Spring data JPA

-> Any java project contains multiple classes 

-> and all the classes will comminicate or talk with each other in the following ways.

	1) Inheritance   - extending & creating objects direcly
	2) Composition   - but creating objects direcly

-> In both the cases the classes become tightly copuled. Why becuase of we are creating Manually one class object in other class Directly.

-> We can take the help of Spring core to create objects and manage them among the classes.

-> To work with Spring Core module we need to ensure best practices to achive the loosly coupling.

-> Spring core module recommends or suggests to use a best approach called "Strategy design pattern" to develop any spring application.


--------------------------------------------------------------------------------------------

"Strategy design pattern"
==========================

-> It's a design pattern comes under Behavioral design pattern.

-> if multiple logics are there it will pick or select one logic at the run time .

 Ex:
 ---
1) Online order ->    pay bill only one mode -> thorough debit

2) Online order -> 	pay bill with multiple modes -> 
						Debit card
						Credit card
						 NetBaning ,
						 Cash on delhivery ,....

Rules : 
-------

-> Favour to Composition instead of Inhertiance.    // still we have problem -> Tightly coupled

-> to Achieve Loose coupling among the classes , Code to the interfaces not to the classes

-> It picks the algorithm or the business logic at the run time of the application.

-> Code should be open for Extension , But closed for Modification.

   [Adding/Deleting functionlities can be applicable , but Modifying the existing code is not applicable]

Inheritance drawbacks
=====================

1) Multiple inheritance is not supported
2) classes will get tightlt coupled


Tightly coupling :
=================

-> If there is changes happend in a one class it will effect another class, it means the 2 classes are said to be tightly coupled.

-> one class will directly dependant on another class.

-> In the real time there are > 100 of classes will be availabe so we need to ensure loosly coupling among the multiple classes.

-> A class contains another classes objects as in the form of instance variables.

public class Car {

	// engine , Wheel , Repairs class instance variables
	public Engine engine;
	public Wheels wheel;
	public Repairs repair;

	public void communicate() {

		engine = new Engine();
		engine.engineStart();

		wheel = new Wheels();
		wheel.wheelsRotate();

		repair = new Repairs();
		repair.repairCar();

	}

	public static void main(String[] args) {
		Car c = new Car();
		c.communicate();

	}

}

Dependancy injection :   // Object injection 
========================

-> The process of Injecting one class object into the another class is called as "Dependancy Injection".

-> Total there are 3 types of dependancy injections are available.


3 types
=======
1) SetterMethod injection 
2) Constructor injection 
3) field level injection
----------------------------------------------------------------------------------------


Before Interface
----------------

BillCollector  <<<<   Multiple Payment options


After Interface
---------------

BillCollector  >>>>   Interface    <<<<  Multiple payment options


---------------------------------------------------------------------------------------------


1) SetterMethod injection 
-----------------------------

-> The process of Injecting one class object as a parameter of setter method is called as Setter method injection.

 Ex : 

		BillCollector bcl = new BillCollector();
		
			bcl.setPayment(new DebitCard());     // injection of onject 


2) Constructor injection 
--------------------------
-> The process of injecting one class object as a parameter for the Constructor is called as 
Constrctor Injection.

Ex : 
		BillCollector bcl  = new BillCollector(new CreditCard());
		bcl .collectPayment();

3) Field Injection
------------------------

-> The process of injecting one class object to the field or variable directly , that is called as Field level injection

Ex: 

		// Field injection
		BillCollector bc3 = new BillCollector();

		bc3.payment = new UpiPayment();		// injecting object to the variable
		bc3.collectPayment();
