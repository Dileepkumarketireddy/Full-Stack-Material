		Spring web MVC
====================================================

-> Spring MVC [ Model View Controller ] 

-> It is just one of the Module in the spring frame work to develop MVC based web applications.

-> We can develop the following type applications.

	1) Web Applications
	2) Distributed Applications [ Restful web services ]

Web Application:
----------------

	Customer -> to Business Communication 
	
	C 2 B [ Customer to Business ]
	---

-> Customer /  User can able to access web application directly through a User Interface 

	Examples: Email , facebooK, Instagram

Distributed Application:- 
-------------------------

-> If one application is communicating with the another application so then we can call it as what 
	Distributed applications

	B 2 B [ Business to Business ]
	----- 
-> These applications we can develop by using 2 ways
	
	-> SOAP web services   [Simple object access protocol]
	-> Rest web services   [Representational state transfer]

-> Note: We can develop both types of services by using Spring MVC module.


	Ex:
	---	Zomato ------ Banking Applications
		RedBus ------ IRCTC, APSRTC, TSRTC ...
		MakeMyTrip ---- IRCTC, ....etc
		Amazon Pay, GPay ---- Banking applications.

Spring web MVC Architecture:
===========================

1) Dispatcher Servlet************************************	
2) Handler Mapper -> find out the controller
3) Handler Adapter  -> to execute the controller 
4) Controller***********************************************
5) Model and View
6) View Resolver*******************************************
7) View


Dispatcher Servlet:
-------------------

-> Dispatcher servlet is a pre-defined class in spring web MVC.
-> This is responsible to perform Pre-processing and post-processing for each and every requests which are coming from end user/ client.
-> it is acts like a Front controller of a web MVC application.

Note: To know the controller class name which is associated/mapped/should be processed with the given Request URL.

-> Dispatcher servlet dispatches the above task to Handler mapper.
	so it will take the help of Handler mapper to do request mapping.

Handler Mapper:
---------------
-> It's a pre-defined class in the spring web mvc.
-> It is used to identify Request handler [Controller request mapping].

-> It will perform or identify the controller class which is going to process the incoming request. by doing Request Mapping with the controller.


Handler Adapter:
----------------
-> Dispatcher servlet is taking a help of Handler Adapter to execute the Controller.
-> So Handler Adapter will call the controller class to execute business logic in my service layer.

Controller:
-----------
-> Its a class which contains a logic to execute the business logic in service layer.
-> Also called as a Request Handler.
-> we can create a controller class by using @Controller or @RestController
-> It will return the data in the form of Model and view object to Dispatcher servlet through Handler adapter

Model and View:
---------------
-> Finally controller executes the business logic which is present in service
   and the final result/response should be kept in the form of "Model and View".

--> Model : The Data should be represents in the from of "Key-value" pair 
--> View : It will return a view logical [file name].
--> To display the data in view file we will use Model view object.

View Resolver:
--------------
-> Dispatcher servlet dispatches the task of Resolving the View name to View resolver
-> used to identify where exactly the view files are physically available in project.

View:
-----
-> View renders the data [ Model ] and returns the response/result.
-> View component renders Model data on physical view file to display that to the end user or client.

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.servlet.ModelAndView;

@Controller
@RequestMapping("/controller1")
public class FirstController {

	@RequestMapping(value = "/one", method = RequestMethod.GET)
	public ModelAndView FirstMethod() {
		ModelAndView m1 = new ModelAndView();
		m1.addObject("message", "data inserted using first Controller 1st method");
		m1.setViewName("index");
		return m1;
	}


	@RequestMapping(value = "/two" , method = RequestMethod.GET)

		ModelAndView m2 = new ModelAndView();

		m2.addObject("message", "data inserted thorugh 1st Controller 2nd METHOD");
		m2.setViewName("index");

		return m2;
	}

}

package springwebmvc.java.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.EnableWebMvc;
import org.springframework.web.servlet.view.InternalResourceViewResolver;

@EnableWebMvc
@ComponentScan(basePackages = "springwebmvc.java.controllers")
@Configuration
public class JavaConfig {

	 @Bean
	    public InternalResourceViewResolver viewResolver() {
	        InternalResourceViewResolver resolver = new InternalResourceViewResolver();
	        resolver.setPrefix("/WEB-INF/views/");
	        resolver.setSuffix(".jsp");
	        return resolver;
	    }
}
