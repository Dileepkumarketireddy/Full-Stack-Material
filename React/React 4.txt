
PROPS:
======
-> props are nothing but properties
-> used to transfer the data from parent component to the child component

	
		Parent --------------->  Child


Ex :
---

````````````Parent sending data - after importing


  	<MyComponent   name='Reactjs'   age='25'   />


`````````````Child receiving data from parent  


 	<h1>my name is : {drops.name} </h1>
        <h1>my age is : {drops.age}</h1>

Output:

my name is : Reactjs
my age is : 25

Notes:
------

-> The values passing from the parent has to enclose either with 'String format' ,  ' 'or close the value with { }.

-> Only premitives we can use either ' ' or { } 

-> For Non Premitive and for function type values alwasy use ->  { } 
 
-> while working with PROPS without de-structure [means -> taking one referance object name as a property] -> we should not enclose the property with { }.

-> We can use any name as a property referance.

  Ex  :    function  MyProperties(props){    }				or
	   function  MyProperties(drops){    }   // any name we can take

           function  MyProperties({props}){   }   // It wont work

-> we can pass any type of data from the parent to child
    EX:  numbers , boolean , Strings, functions , arrays , objects .

-> Always it's a parent to the child relation  ,No it's not possible to modify the data from the child [Read Only].

-> We Can perform DeStructuring as well

Default values
==============

-> When a Child component function accept some parameter values , but parent faild to send the values , in this case we can use these default values approach to intialise the parameters with default values [backup values].

Note :
-----

Always enclose the parameters and their values with in { } curly brackets , 
even it contains only one or more than one


->   Ex :

	assign the value directly inside the function parameters

```````````function DefaultValues({ name = 'DELL', price = 77887}) {

	    return (
        	<>
            <h1> name of device : {name}</h1>
            <h1>price of device : {price}</h1>
		</>
		);
	}


Output :

name of device : DELL
price of device : 77887

------------------------------------------------------------------------------------

-> We can use these Default values in 2 cases 

	1) Function components
	2) Class components

JSON :
======

->JAVA SCRIPT OBJECT NOTATION

-> It represents the data in the form of key's and value's , and used to trnasfer the data between the applications

-> Convertiosn process from Java script object to JSON object vice versa.

-> Always keys should represent with double quote -> " keys " -> String format

-> Always values should represent with double quote -> " values " -> String format -> Except Numbers and booleans.

FORMAT OF JSON
---------------


{
	"product":"apple",
	"model":"17pro",
	"price":"1L",
	"rating":4.8,						// number
	"isExpensive":true					// boolean
	"availableIn":["india","canada", true , false]		// Array type
	"details":{"screen":"amoled","stock":999,"ServiceCenter":true}  //nestedObj
}


1) Java script to JSON :
-------------------------

-> To Convert Java script object into JSON object , there is a method called JSON.stringfy().

-> Ex : 		
	
// Normal Java script object

var data = {

    product: 'apple',
    model: '17pro',
    price: '1L',
    rating: 4.8,
    isExpensive: true,
    availableIn: ['india', 'china', 'usa', 'canada' ,124 , true , false ],
    details : {
        screen : 'amoled',
        stock : 999,
        ServiceCenter : true
    }

};

// Converting into JSON object
var result = JSON.stringify(data);

console.log(result);



Note :
-----

-> If any value in java script object is function type , it will not convert to json object

Output :
-------
{
	"product":"apple",
	"model":"17pro",
	"price":"1L",

	"isExpensive":true					// boolean
	"availableIn":["india","canada", true , false]		// Array type

-> There are some features are available in React.
	
	Ex : States, Life cycle methods , Effects,  .....

-> Before 16.8v To use or implements or work with the above features  , Always its only possible with the help of Components created with the Classes.

-> To Enable these features to work work with Functional components one approch introduced , that approach is called HOOKS.

-> Hooks are like a predefind Functions used to implements react features with Functional components.


========
Hooks :
========

->usestate()


Syntax  :

	const[ variable1  ,  setVariable1 ]   =   usestate(value);	



variable1   -> what ever the value we are passing as a parameter for usestate(value)	.


	EX :

		usestate(1);		->>>>>    variable1 =  1
		usestate(5);		->>>>>    variable1 =  5 



setVariable1   -> used to To update or modify the variable1
